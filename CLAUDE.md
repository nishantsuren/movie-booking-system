## Conventions
- **DB**: `psycopg2` only, no ORM. Flag before switching.
- **Migrations**: `infra/migrations/NNN_*.sql`, applied by script, tracked via `schema_migrations`. No framework.
- **Service structure**: catalog + theatre use `admin/`/`customer/`/`common/` sub-packages (v20). Add routes to the right sub-package, never to `main.py`. Shared config including `AUTH_ENABLED` → `common/config.py`.
- **Config**: every service has its own `config.py` (v22) — `common/config.py` for catalog/theatre, top-level `config.py` for booking/payment/user/routing/local-cdn-mock/agent-service. All env-derived values and magic numbers (TTLs, retry/backoff/batch-size knobs, advisory lock keys, external URLs) live there, never inline in the module that uses them. Pure leaf module per service (only `import os`) — anything can import from it with no circular-import risk. Booking's per-worker constants are prefixed (`RECONCILIATION_*`/`OUTBOX_RELAY_*`/`AVAILABILITY_SYNC_*`) since the three workers had same-named-but-different-valued constants before centralizing. Gotcha (v23): a bare top-level `config.py` per service means two services' `config` modules collide under the same unqualified name if both get pulled into one pytest process via `sys.path` hacks — see Testing below.
- **Agent service**: `services/agent-service/`, :8007, registered in `scripts/dev.sh` + routing's `SERVICE_MAP` under prefix `agent`. Routes are defined *without* their own `/agent` prefix internally (`/message`, `/session/{id}` in `main.py`) — routing adds it back, same convention as every other service. Option B architecture (design doc / `docs/ai-agent-requirements.md`): `nlu.py` calls Ollama at most once per turn for intent+entities JSON only, never decides/responds/calls APIs; `dialogue_manager.py` owns every decision, every platform call, every response (from `templates.py`, never generated). Python 3.9 like everything else — `X | None` annotations need `from __future__ import annotations`.
- **Idempotency**: `shared/idempotency/idempotency.py` — `INSERT...ON CONFLICT`, server-derived hash key. See `_derive_idempotency_key` in catalog/theatre `common/`. Design §11.1.
- **Auth**: `shared/auth/auth.py` — `get_auth_context`/`require_role(...)` on every endpoint. `AUTH_ENABLED` toggle (§3.2).
- **Events**: `shared/events/events.py` — `LoggingEventPublisher` no-op until Phase 13.
- **TheatreIntegration**: `MockTheatreIntegration` always wired locally/in tests. `THEATRE_MOCK_HOLD_MODE`/`THEATRE_MOCK_CONFIRM_HOLD_FAILS` env vars trigger failure modes for tests against the live process. Never call a real theatre API locally. `confirm_hold` is synchronous-first in `confirm()` (v21) — Outbox is fallback-only; `release_hold` (cancel/sweep) stays purely async, deliberately asymmetric. Design §5.7.
- **Testing**: DB tests hit real Dockerized Postgres, never mocked. Pattern: `shared/tests/`. If a test file does `sys.path.insert(...)` to reach into a service's own bare-named modules (e.g. `config`, several services have one) for a module-level import, save and restore those `sys.modules` entries around the import — pytest's single up-front collection pass imports every test file before any test body runs, so a module-level import in one file can poison a same-named module another file imports later, *inside a test function*, regardless of which file's tests actually execute first (hit between `test_phase5.py` and the new agent-service tests' shared `config` name, v23).
- **Services**: `docker-compose.yml` = Postgres + Redis only. Run natively via `scripts/dev.sh`. Add new services there. Subcommands for headless/agent-started sessions with no TTY to Ctrl+C: `list` (what's running + pid/port), `kill <name>`, `killall`; default/no-arg is `startall` (unchanged blocking + Ctrl+C behavior). Script must stay bash-3.2-compatible (macOS system `/bin/bash`) — no `declare -A`, use a `case` lookup instead.
- **Frontend**: React+Vite+TypeScript in `apps/<name>/`. API URL via `VITE_API_BASE_URL`. admin-web: `base: '/admin/'` + `assetsDir: 'app-assets'` in vite.config.ts. E2E: Playwright via `npm run test:e2e` against deployed build (`npm run build:deploy`), not dev server. `GET /admin` → `/admin/` redirect in local-cdn-mock — do not remove.
