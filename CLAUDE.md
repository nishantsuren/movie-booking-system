Engineering lessons worth not re-discovering the hard way:
- From the deleted agent build (code is gone; findings were empirically real): FastMCP encodes a *list*-returning tool result as one `TextContent` block **per list element**, not one block for the whole array ‚Äî a naive `content[0]` read silently truncates any list tool to its first item.
- The model calling an MCP tool has no way to know the real session id unless told, and will invent a placeholder rather than fail ‚Äî unconditionally override any model-supplied session/identity argument with the real one before dispatching.
- A tool's raw platform dict (idempotency keys, timestamps, internal flags) can get garbled when a small model summarizes it ‚Äî trim to customer-relevant fields at the source, don't trust the model's judgment about what's internal.
- A small model sometimes narrates a tool call as plain/JSON text instead of using the structured tool-calling field ‚Äî detect generically (does any real tool name appear literally in the text?) and fail closed rather than relay raw JSON to a customer.
- `llama3.2:3b` extracting a bare ordinal ("the first one") succeeded ~80‚Äì90% of the time per call ‚Äî plan for graceful re-prompt on a miss, not a guaranteed first-try parse.
- `search_movies()`-style substring filtering only works if catalog actually has the filterable field (no genre column existed) ‚Äî verify the data model supports a query before promising the agent can answer it.

## Conventions
- **DB**: `psycopg2` only, no ORM. Flag before switching.
- **Migrations**: `infra/migrations/NNN_*.sql`, applied by script, tracked via `schema_migrations`. No framework.
- **Service structure**: catalog + theatre use `admin/`/`customer/`/`common/` sub-packages (v20). Add routes to the right sub-package, never to `main.py`. Shared config including `AUTH_ENABLED` ‚Üí `common/config.py`.
- **Config**: every service has its own `config.py` (v22) ‚Äî `common/config.py` for catalog/theatre, top-level for booking/payment/user/routing/local-cdn-mock/agent-mcp-server/agent-service. All env-derived values/magic numbers (TTLs, retry/backoff/batch-size, lock keys, external URLs) live there, never inline. Pure leaf module (`import os` only) ‚Äî zero circular-import risk. Booking's per-worker constants are prefixed (`RECONCILIATION_*`/`OUTBOX_RELAY_*`/`AVAILABILITY_SYNC_*`; previously same-named/different-valued before centralizing). Gotcha: a bare top-level `config.py` per service means two same-named `config` modules collide if both get pulled into one pytest process via `sys.path` hacks ‚Äî see Testing.
- **Python version**: shared `.venv` is Python **3.12.13**, bumped from 3.9 for the `mcp` SDK's `>=3.10` requirement ‚Äî not 3.14, since `mcp`'s `sse-starlette` dep needs `starlette>=0.49.1` there, conflicting with every service's `fastapi==0.115.0`‚Üí`starlette<0.39.0` pin; `sse-starlette==2.0.0` resolves to `starlette==0.38.6`, satisfying both. `scripts/dev.sh` pins `python3.12` on venv creation (`brew install python@3.12` if missing). One shared venv ‚Äî a new dependency anywhere is worth a `pip check`.
- **Agent**: `services/agent-service/` (:8007) is being rebuilt incrementally ‚Äî see `docs/agent_service_progress.md` for current state. Backend routes never include their own routing prefix (`POST /message`, not `/agent/message` ‚Äî routing's `forward()` strips it; every other service already follows this). `services/agent-mcp-server/` (:8008, untracked): five read-only lookup tools + one stateful `handle_booking_turn` tool meant to wrap agent-service's `POST /internal/handle-turn` ‚Äî that endpoint doesn't exist yet, so this service stays non-functional until built. Both registered in `scripts/dev.sh` + routing's `SERVICE_MAP` under prefix `agent`; `scripts/dev.sh startall` succeeds end-to-end.
- **Idempotency**: `shared/idempotency/idempotency.py` ‚Äî `INSERT...ON CONFLICT`, server-derived hash key. See `_derive_idempotency_key` in catalog/theatre `common/`. Design ¬ß11.1.
- **Auth**: `shared/auth/auth.py` ‚Äî `get_auth_context`/`require_role(...)` on every endpoint. `AUTH_ENABLED` toggle (¬ß3.2).
- **Events**: `shared/events/events.py` ‚Äî `LoggingEventPublisher` no-op until Phase 13.
- **TheatreIntegration**: `MockTheatreIntegration` always wired locally/in tests. `THEATRE_MOCK_HOLD_MODE`/`THEATRE_MOCK_CONFIRM_HOLD_FAILS` env vars trigger failure modes against the live process. Never call a real theatre API locally. `confirm_hold` is synchronous-first in `confirm()` (v21) ‚Äî Outbox is fallback-only; `release_hold` (cancel/sweep) stays purely async, deliberately asymmetric. Design ¬ß5.7.
- **Testing**: DB tests hit real Dockerized Postgres, never mocked (`shared/tests/`). If a test file `sys.path.insert(...)`s into a service's own bare-named modules (e.g. `config`) for a module-level import, save/restore those `sys.modules` entries around it ‚Äî pytest's up-front collection imports every test file first, so one file's module-level import can poison a same-named module another file imports later *inside a test function* (hit once, between `test_phase5.py` and booking's own `config`). No `pytest-asyncio` ‚Äî an async function under test gets a plain sync `def test_...()` wrapper calling `asyncio.run(...)` on an inner async helper, not a `@pytest.mark.asyncio` decorator. Current set: `test_phase1`‚Äì`test_phase7`, `test_phase9_5`, `test_phase9_6_find_seats`, plus agent-service tests (real routing/theatre/agent processes, no mocks ‚Äî same convention as the rest).
- **Services**: `docker-compose.yml` = Postgres + Redis only. Run natively via `scripts/dev.sh`. Add new services there. Subcommands for headless/agent-started sessions with no TTY to Ctrl+C: `list` (what's running + pid/port), `kill <name>`, `killall`; default/no-arg is `startall` (unchanged blocking + Ctrl+C behavior). Script must stay bash-3.2-compatible (macOS system `/bin/bash`) ‚Äî no `declare -A`, use a `case` lookup instead.
- **Frontend**: React+Vite+TypeScript in `apps/<name>/`. API URL via `VITE_API_BASE_URL`. admin-web: `base: '/admin/'` + `assetsDir: 'app-assets'` in vite.config.ts. E2E: Playwright via `npm run test:e2e` against deployed build (`npm run build:deploy`), not dev server. `GET /admin` ‚Üí `/admin/` redirect in local-cdn-mock ‚Äî do not remove.
